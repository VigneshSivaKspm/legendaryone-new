# Email Setup Guide - Legendary One Contact Form

This guide will help you set up Gmail email forwarding for the contact form using nodemailer and Gmail App Passwords.

## Prerequisites

- A Gmail account
- Node.js installed on your system

## Step 1: Enable 2-Factor Authentication on Gmail

1. Go to [Google Account Security](https://myaccount.google.com/security)
2. Sign in to your Google Account
3. On the left, click **Security**
4. Enable **2-Step Verification** if not already enabled

## Step 2: Create a Gmail App Password

1. Go to [Google Account Security](https://myaccount.google.com/security)
2. Click **App passwords** (appears only if 2FA is enabled)
3. Select **Mail** and **Windows Computer** (or your OS)
4. Google will generate a 16-character password
5. Copy this password (you'll need it in the next step)

## Step 3: Update Environment Variables

Open `.env` file and fill in:

```
GMAIL_USER=your-email@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
CONTACT_EMAIL=legendaryoneff@gmail.com
```

**Important:**

- `GMAIL_USER` - Your Gmail address that will send emails
- `GMAIL_APP_PASSWORD` - The 16-character password generated by Google (use spaces as shown)
- `CONTACT_EMAIL` - Where form submissions will be sent (can be same as GMAIL_USER or different)

## Step 4: Install Dependencies

Run in terminal:

```bash
npm install
```

This will install:

- express (backend server)
- nodemailer (email service)
- cors (cross-origin requests)
- dotenv (environment variables)
- concurrently (run multiple commands)

## Step 5: Start Development

### Option A: Run Frontend and Backend Together

```bash
npm run dev:all
```

This runs:

- Backend server on `http://localhost:5000`
- Frontend on `http://localhost:5173` (or default Vite port)

### Option B: Run Separately (for debugging)

**Terminal 1 - Backend:**

```bash
npm run server
```

**Terminal 2 - Frontend:**

```bash
npm run dev
```

## Step 6: Test the Contact Form

1. Open your site at `http://localhost:5173`
2. Navigate to the **Contact** page
3. Fill out the form and submit
4. You should receive two emails:
   - Confirmation email to the user
   - Submission notification to your contact email

## Troubleshooting

### "Failed to send email" error

1. Check if Gmail credentials in `.env` are correct
2. Ensure 2FA is enabled on your Gmail account
3. Verify the App Password is correct (no typos)
4. Check that backend server is running on port 5000

### Backend won't start

1. Check if port 5000 is already in use
2. Change `PORT` in `.env` to a different number (e.g., 3000)
3. Restart the server

### CORS errors

Make sure you're using the correct endpoint:

- Development: `/api/contact` (relative path works locally)
- Production on Vercel: `/api/contact` (same path on Vercel)

### Email not sending

1. Check backend/serverless function logs
2. Verify Gmail App Password (spaces included)
3. Check Vercel function logs if deployed
4. Try regenerating the App Password

## Security Notes

⚠️ **Never commit `.env` file to git** - it contains sensitive information

- `.env` is already in `.gitignore`
- Never share your App Password
- Use environment variables in Vercel Dashboard for production

## For Production Deployment (Vercel)

For detailed Vercel deployment instructions, see **VERCEL_DEPLOYMENT.md**

Quick steps:

1. Push code to GitHub
2. Connect repository to Vercel
3. Set environment variables in Vercel Dashboard:
   - `GMAIL_USER`
   - `GMAIL_APP_PASSWORD`
   - `CONTACT_EMAIL`
   - `VITE_GOOGLE_API_KEY`
   - `VITE_GOOGLE_PLACE_ID`
4. Vercel automatically deploys and builds
5. Your serverless functions are live!

## Email Templates

Emails are sent with:

- **To Business:** Full submission details
- **To User:** Confirmation message

Templates are customizable in:

- Development: `server.js`
- Production: `api/contact.js`

Look for the `transporter.sendMail()` calls to customize.
